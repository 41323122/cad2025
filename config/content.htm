心得

當初建立 Team 專案時，我以為只要大家建立好帳號、開好分支，剩下的就只是把程式碼丟上去就好。沒想到真正開始合併時，才發現後面有一堆事情要處理。

我們遇到最大的問題是多人 Git Pull 時，組員間的資料無法同步。後來才發現，原因出在我們太心急，沒有先拉取（Pull）最新的進度就直接進行編輯。這導致大家在不同的版本上工作，合併時出現了許多錯誤和衝突。這次經驗讓我深刻體會到，Git 協作不只是寫程式，更是一種邏輯的同步。雖然遇到很多錯誤，但現在我終於知道「先拉再寫」才是團隊開發最重要的規矩。

w10: Webots 是什麼？有什麼特點？ 如何進行場景模擬？ 如何用 Python 進行控制系統設計？ (Tutorial)

Webots 是什麼？ Webots 是一款基於 ODE 物理引擎的專業級開源機器人模擬軟體，能精確模擬重力、摩擦力及感測器回饋，是實現機器人數位孿生（Digital Twin）的核心工具。

核心特點： 具備強大的跨平台多語言支援（C++/Python/Java/ROS）與豐富的組件庫，能將設計好的 CAD 模型賦予真實物理碰撞特性與動力學反應。

場景模擬流程： 透過 Scene Tree（場景樹） 階層式架構，依序添加 WorldInfo（物理環境）、Viewpoint（視角）、地圖與機器人節點，並為機器人定義 boundingObject（碰撞體）與 Physics（質量分配）。

Python 控制系統設計： 利用 Webots API 初始化機器人實例，並在一個以 robot.step(timestep) 為核心的 while 迴圈中，即時讀取感測器數值並計算運動學指令，最後透過 motor.setPosition() 或 setVelocity() 驅動關節。

w14: Webots 的模型有沒有辦法用程式方法建立? 有沒有辦法直接從 CAD 套件中轉出 Webots 的場景模型？

1. Webots 模型可以用程式方法建立嗎？

可以，主要透過以下兩種方式：

  • 使用 PROTO 檔案（參數化建模）： Webots 的 PROTO 是一種強大的腳本化節點。你可以撰寫一個 .proto 檔案，將機器人的腿長、寬度、馬達參數設為變數。這樣一來，只需更改一個參數，Webots 就會自動重新計算並生成對應的物理模型，非常適合用於 5-bar Linkage 的尺寸優化。

  • 動態節點操作（API）： 透過控制程式（Controller），可以使用 Python API（如 root_children_field.importMFNodeFromString()）在模擬運行時，動態地「刷」出牆壁、障礙物或零件。

2. 能否直接從 CAD 套件轉出 Webots 模型？

可以，這是目前開發 OpenDuck Mini 最推薦的作法：

  • URDF 導出（最常用）： 大多數工業級 CAD（如你使用的 Onshape、SolidWorks、Fusion 360）都有開源插件（如 onshape-to-robot）。這些插件能將 CAD 中的馬達軸心、零件重心、慣性矩與視覺模型整合，導出為 URDF 格式，而 Webots 可以直接匯入 URDF 並自動生成機器人結構。

  • Webots Importer 支援： Webots 內建了多種匯入器，支援將 DAE (Collada)、OBJ 或 VRML 格式轉入。

    • DAE 格式： 適合保留 CAD 中的精緻視覺外觀。

    • STL 格式： 通常用於定義 boundingObject（碰撞體），因為它結構簡單，物理計算效率最高。
